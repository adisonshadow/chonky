
路线图分为两大阶段：
- **第一阶段（MVP）：** 基于 npm 包（Babel/SWC 插件）实现核心语法与元数据收集，快速验证设计理念。
- **第二阶段（性能跃升）：** 采用 Rust 重构编译器核心，实现自主性能调优与极致编译速度。

每个阶段内部均以 **规范与设计文档** 作为起始里程碑。

---

### Chonky 实施路线图与里程碑（基于 v1.1 设计）

#### 第一阶段：基于 npm 包的功能验证与生态集成

**总体目标：**  
在不动摇现有 TypeScript 编译器生态的前提下，通过一组 npm 包（编译插件 + 运行时库 + CLI 工具）实现 Chonky 的核心语法扩展、元数据收集协议及人机协作基础流程，使开发者能够立即在现有项目中试用 Chonky 模式。

##### 里程碑 1.1：核心规范与接口文档冻结

- **产出物：**
  - 《Chonky 结构化需求定义规范 v1.0》：详细定义 `defineRequirement` 的 JSON Schema 字段含义、类型约束及示例。
  - 《Chonky 渲染元数据协议 v1.0》：明确 `window.__CHONKY_RENDER_META__` 的数据结构、上报时机及查询接口规范。
  - 《Chonky 验证规则引擎规范 v1.0》：定义 `machine:assert` 的语法扩展规则、编译目标（Jest/Vitest）及与需求 ID 的绑定方式。
  - 《Chonky 歧义消解协议 v1.0》：说明 `negate` 字段的使用规则及编译时双向校验的触发逻辑。
  - 《Chonky 静默模式与交互式询问阈值规范 v1.0》：列出各类资源优化的默认阈值、静默触发条件及终端询问模板。

- **验收标准：** 所有规范文档通过内部评审，作为后续工具链实现的唯一依据。

##### 里程碑 1.2：Babel/SWC 转译插件开发

- **目标：** 实现 Chonky 扩展语法的编译时转换，将 `.chonky.ts` / `.cts` 文件转换为标准 TypeScript 代码。
- **关键任务：**
  - 开发 `@chonky/babel-plugin` (或 SWC 插件)，识别并处理以下语法节点：
    - `defineRequirement(...)` → 剥离包装，输出纯净对象，并生成独立 JSON 清单文件。
    - `machine:assert for "ID" { ... }` → 转换为 Vitest/Jest 测试用例代码。
    - `with moduleCalls { ... }` → 保留为注释或元数据，供静态分析工具读取。
  - 实现 JSX 组件自动包裹逻辑，注入渲染元数据收集钩子（开发模式专用）。
  - 提供 Source Map 支持，确保编译后代码可调试。
- **产出物：** `@chonky/transpiler` npm 包，包含插件本体及配套的类型定义。

##### 里程碑 1.3：运行时库与浏览器端基础设施

- **目标：** 提供浏览器端运行所必需的基础功能，支撑元数据收集、验证执行及优化组件。
- **关键任务：**
  - 开发 `@chonky/runtime` 包，包含：
    - `ChonkyRenderer` 类：负责收集并维护 `__CHONKY_RENDER_META__` 数据。
    - `Image` 组件：集成资源优化决策逻辑，根据配置读取静默模式阈值。
    - `verify` 运行时辅助函数：用于开发环境下的断言执行。
  - 开发浏览器开发者工具面板（Chonky DevTools），可视化展示渲染元数据、依赖图谱及需求完成度。
- **产出物：** `@chonky/runtime` 和 `@chonky/devtools` npm 包。

##### 里程碑 1.4：CLI 工具与静态分析器

- **目标：** 提供命令行入口，封装构建流程、依赖图生成及优化询问交互。
- **关键任务：**
  - 开发 `@chonky/cli`，提供以下命令：
    - `chonky init`：交互式初始化 Chonky 项目（生成 `chonky.config.js`、`tsconfig` 扩展、目录结构及示例 `.req.ts` 文件），支持 AI Agent 以 `--yes` 静默模式直接调用。
    - `chonky dev`：启动开发服务器，注入运行时库并启用元数据收集。
    - `chonky build`：执行生产构建，自动剔除所有调试代码（Tree-Shaking）。
    - `chonky graph`：调用静态分析器（基于 TypeScript Compiler API 或 madge）生成模块依赖图 JSON。
    - `chonky optimize`：交互式扫描项目资源，按静默模式阈值询问并执行优化。
  - 集成 Webpack / Vite 插件，降低与主流构建工具的集成门槛。
- **产出物：** `@chonky/cli` npm 包，以及对应的 Vite/Webpack 插件。

##### 里程碑 1.5：语义化翻译视图（人类兜底机制）

- **目标：** 实现“机器优先代码”与“人类可读视图”的实时双向转换，保障紧急人工介入可行性。
- **关键任务：**
  - 开发 `chonky view` 子命令，读取 Chonky 源文件并输出美化、高亮、带语义命名的 TypeScript 等效代码。
  - 开发 `chonky revert` 子命令，允许开发者将修改后的语义化代码反向编译为机器优化格式（基于 AST 匹配）。
- **产出物：** CLI 扩展功能模块，配套使用文档。

##### 里程碑 1.6：集成测试与示例项目

- **目标：** 验证第一阶段所有组件协同工作，并提供可复现的演示案例。
- **关键任务：**
  - 编写端到端测试用例，覆盖从需求定义到渲染元数据上报的完整链路。
  - 构建一个名为 `Chonky TodoMVC` 的示例应用，完全使用 Chonky 语法编写，并展示需求追溯、元数据调试面板等功能。
- **产出物：** 测试报告、示例仓库及对应的在线演示环境。

#### 第二阶段：Rust 编译器核心与自主性能调优

**总体目标：**  
用 Rust 语言重写 Chonky 编译器核心，替换 Babel/SWC 转译环节，实现编译速度的数量级提升，并在此基础上开启“自主性能调优”（自动 Wasm 拆分、智能合成层分配等高级优化）。

##### 里程碑 2.1：Rust 编译器架构设计文档

- **产出物：**
  - 《Chonky 编译器（Rust）架构设计书》：包含 AST 定义、解析流程、类型检查集成方案（与 TSC 的协作模式）、增量编译策略。
  - 《Chonky 自主性能调优算法规范》：明确自动函数拆分的判定条件、Wasm 转换触发阈值、合成层分配启发式规则。
- **验收标准：** 文档经过技术评审，明确 Rust 编译器与现有 TypeScript 生态的兼容边界。

##### 里程碑 2.2：Rust 解析器与基础转译器实现

- **目标：** 实现 `.chonky.ts` 文件的解析与基础语法转换，达到与第一阶段转译插件等价的功能。
- **关键任务：**
  - 使用 `swc_ecma_parser` 或 `oxc` 构建 AST 解析器。
  - 实现 `defineRequirement`、`machine:assert` 等扩展语法的 AST 转换逻辑。
  - 通过 FFI 或语言服务协议与 TypeScript 类型检查器交互，确保类型安全。
- **产出物：** 可执行的 Rust 二进制 `chonkyc`，能够将 Chonky 源码编译为标准 TypeScript 代码。

##### 里程碑 2.3：极致性能优化模块开发

- **目标：** 实现第一阶段无法完成的编译时自主优化能力。
- **关键任务：**
  - **自动函数拆分：** 基于 AST 复杂度分析，标记长耗时函数并自动生成 Web Worker 或 Wasm 模块包装代码。
  - **智能合成层分配：** 在编译 JSX 时，分析动画相关 CSS 属性，自动注入 `will-change` 及合成层提升代码。
  - **静态节点提升 2.0：** 识别并提升静态闭包函数，减少运行时内存分配。
- **产出物：** 集成上述优化的 Rust 编译器版本，并提供对应的配置开关。

##### 里程碑 2.4：工具链整合与迁移指南

- **目标：** 将 Rust 编译器无缝接入现有的 CLI 工具和构建插件体系，确保用户平滑升级。
- **关键任务：**
  - 修改 `@chonky/cli`，使其能够自动检测并调用 Rust 编译器后端（通过 Node-API 或子进程）。
  - 编写《从第一阶段迁移至第二阶段指南》，说明配置变更及潜在破坏性变化。
- **产出物：** 新版 CLI 与构建插件，以及完整的迁移文档。

##### 里程碑 2.5：性能基准测试与正式发布

- **目标：** 量化第二阶段编译器带来的性能提升，并正式发布生产可用版本。
- **关键任务：**
  - 构建包含 10,000+ 模块的大型模拟项目，对比第一、二阶段的冷/热编译时间。
  - 在真实世界应用中测量自动 Wasm 拆分带来的首屏加载优化效果。
- **产出物：** 性能白皮书、Chonky v2.0 正式版发布公告。

### 路线图总览图（阶段-里程碑依赖关系）

```
第一阶段 (npm 包)
├─ 1.1 规范文档 ◄── 前置条件
├─ 1.2 转译插件 ──┐
├─ 1.3 运行时库 ──┤ 可并行
├─ 1.4 CLI 工具 ──┘
├─ 1.5 语义化视图
└─ 1.6 集成测试

第二阶段 (Rust 编译器)
├─ 2.1 架构设计 ◄── 前置条件
├─ 2.2 解析与基础转译
├─ 2.3 极致性能优化
├─ 2.4 工具链整合
└─ 2.5 性能基准与发布
```

此路线图确保每个阶段均以明确的文档产出作为起点，后续实现严格遵循设计规范，并预留了从轻量级 npm 包向高性能 Rust 编译器演进的清晰路径。
